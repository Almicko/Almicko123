1. Три области Git:

Рабочая папка - файлы на диске

Индекс (staging) - подготовленные к коммиту изменения

Репозиторий - история коммитов (версии)

2. Команды:

git add - добавляет изменения в индекс

git commit - фиксирует изменения из индекса в репозиторий

git push - отправляет коммиты в удалённый репозиторий

3. Сообщения коммитов должны объяснять что и почему изменено, а не просто "исправление". Это помогает понять историю изменений.

4. git add имя_файла vs git add .:

git add имя_файла - добавляет конкретный файл

git add . - добавляет все изменённые файлы
Лучше использовать выборочное добавление для контроля над тем, что попадает в коммит.

5. git status показывает:

Изменённые файлы

Файлы в индексе (готовые к коммиту)

Неотслеживаемые файлы

6. git remote add origin URL связывает локальный репозиторий с удалённым. "origin" - стандартное имя для удалённого репозитория.

7. -u в git push -u origin main устанавливает связь между локальной и удалённой веткой, чтобы в дальнейшем использовать просто git push.

8. Ветка по умолчанию - main (ранее master). Это основная ветка разработки.

9. git log показывает историю коммитов: хеш, автора, дату, сообщение.

10. Если не сделать git add перед git commit - изменения не попадут в коммит, так как они не были добавлены в индекс.
