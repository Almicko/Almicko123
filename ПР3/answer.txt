1. ID селектор (#id) имеет высшую специфичность (0,1,0,0), класс (.class) — среднюю (0,0,1,0). ID должен быть уникальным, классы — многократно переиспользуемые.

2. Каскад учитывает: 1) источник стилей, 2) специфичность, 3) порядок. При конфликте побеждает правило с высшей специфичностью, при равенстве — последнее объявленное.

3. Специфичность:

ul li.important = 0,0,1,2 (2 элемента + 1 класс)

#site-title = 0,1,0,0 (1 ID)

.content > .featured-post = 0,0,2,0 (2 класса)

4. Наследуются: color, font-family, line-height. Не наследуются: margin, padding, border.

5. > выбирает только прямых потомков, пробел — любых вложенных.

6. :hover — стили при наведении, :nth-child() — выбор элементов по порядку. Используются для интерактивности и таблиц/списков.

7. !important принудительно применяет стиль, игнорируя специфичность. Ломает естественный каскад, усложняет поддержку.

8. ::before/::after создают псевдоэлементы внутри тега. Могут содержать текст, иконки, декоративные элементы через content.

9. При одинаковой специфичности побеждает последнее правило в CSS-файле.

10. Инструменты разработчика показывают:

применённые стили

вычисленные значения

каскадную модель

позволяют редактировать стили в реальном времени

показывают наследование
